## 二、考点

- **知识点1** 在数据结构中，从逻辑上可以把数据结构分为线性结构和非线性结构

线性结构指的是数据元素中存在一对一的节点，只有一个开始的节点和一个终端节点。

 **线性表、栈、队列是比较常见的线性结构。**

  非线性结构指的是数据元素中存在一对多或者是多对多的关系，存在多个前驱或者是后继。
 **树、图、集合、多维数组、排序、查找、广义表是比较常见的非线性结构。**
  > 在数据结构中，前驱和后继是指在某种数据结构中，一个元素的前面和后面紧挨着的元素。具体来说，前驱是指在某个线性结构（如链表）中，一个节点的前一个节点；后继是指一个节点的后一个节点。这种关系在双向链表中尤为常见，每个节点都包含一个数据元素和指向前一个节点和后一个节点的指针。

  > 非线性结构中的前驱和后继有以下区别：
  >
  > - 前驱：在非线性结构中，前驱是指一个元素在结构中的直接前面的元素。换句话说，前驱是指一个元素的直接上一个元素。非线性结构中的前驱可以有多个，因为一个元素可能有多个直接前驱。
  > - 后继：后继是指一个元素在非线性结构中的直接后面的元素。后继是一个元素的直接下一个元素。与前驱类似，非线性结构中的后继也可以有多个，因为一个元素可能有多个直接后继。
  >

- **知识点2** 算法分析的两个主要方面是时间复杂性和空间复杂性

- **时间复杂性** ：算法都是由控制结构对数据类型的整体操作构成的，算法的运行时间是算法所有语句执行时间的总和。
  时间复杂度取决于问题的规模和待处理数据的初态。如排序算法的时间复杂度是和数据的
- **空间复杂性** ：算法转换为计算机程序执行过程中所需的空间。

  一般时间复杂度与空间复杂度统称为算法复杂度。

- **知识点4** 在长度为 `n` 的顺序表的第 `i(1 ≤ i ≤ n+1)`个位置上插入一个元素，元素的移动次数为 `n-i+1`

> `i(1 ≤ i ≤ n+1)` 是一个数学中的范围表示，用来描述一个变量 i 的可能取值范围。
具体来说：
`1 ≤ i` 表示 `i` 的值至少为 1。
`i ≤ n+1` 表示 `i` 的值最大为 `n+1`。
所以，结合起来 `i(1 ≤ i ≤ n+1)` 表示 i 的取值范围是从 1 到 `n+1`，包括 1 和 `n+1`。
例如，如果 n = 5，那么 i 的可能取值为：1, 2, 3, 4, 5, 6。

例如：`n = 4`
当前顺序表为：`| 1 | 2 | 3 | 4 |`。当 `i = 3`时，在`第3位`插入`元素 5`，顺序表变为：`| 1 | 2 | 5 | 3 | 4 |`。这意味着，`元素 5` 需要将原来的`第4位（元素4）`向后移动 1 位，将原来的`第3位（元素3）`向后移动 1 位，才能在 3 位上插入元素 5。
也就是 `4 - 3 + 1 = 2`的移动次数。

- **知识点6** 在向顺序栈中压入元素时先移动栈顶指针，后压入元素

**“先移动栈顶指针，后压入元素”**意味着在执行插入操作前，首先需要更新栈顶指针的位置，然后再将新元素放入该位置。具体步骤如下：

1. **移动栈顶指针**：通常，栈顶指针会向上移动，指向下一个可用的位置。在数组实现中，这通常意味着增加栈顶指针的值。
2. **压入元素**：将新元素放入栈顶指针所指向的位置。这样做的目的是为了确保新元素能够被正确地插入到栈中，并且栈顶指针始终指向栈顶的元素。
3. **更新栈顶指针**：在压入元素之后，栈顶指针需要被更新，使其指向新的栈顶元素。

- **知识点8** 稀疏矩阵一般的压缩存储方法有两种，即三元组和十字链表

- 三元组是一种数据结构，用来存储稀疏矩阵。我们可以把稀疏矩阵想象成一个有很多0的数表。为了节省存储空间，我们只需要存储非零元素的信息。三元组就是用来存储这些非零元素的信息的数据结构。
具体来说，一个三元组包含三个信息：行标、列标和值。行标表示这个非零元素在矩阵的第几行，列标表示这个非零元素在矩阵的第几列，值就是这个非零元素的值。
例如，假设有一个3x3的稀疏矩阵如下：
1 0 0
0 2 0
0 0 3
我们可以用三个三元组来存储这个矩阵的信息：(1,1,1)、(2,2,2)、(3,3,3)。
- **十字链表是一种数据结构，用来存储有向图。**
  十字链表通过将每个顶点的出边和入边分别用链表连接起来，实现了同时获取顶点的出度和入度的目的。每个顶点在十字链表中对应一个结点，结点中包含了顶点的数据以及指向该顶点的出边链表和入边链表。
  通过使用十字链表，我们可以更高效地存储和操作有向图的信息。它可以帮助我们快速计算顶点的出度和入度，进行图的遍历和搜索，以及解决一些与有向图相关的问题。
  在十字链表中，每个顶点都有一个对应的表头结构，其中包含了以下几个字段：
  - row：表示顶点所在的行号。
  - col：表示顶点所在的列号。
  - next：指向下一个具有相同行号的顶点的指针，用于构建行链表。
  - right：指向下一个具有相同列号的顶点的指针，用于构建列链表。
  - down：指向该顶点的下一个顶点的指针，用于构建邻接表。
  非零元素节点结构：
  - row：表示顶点所在的行号。
  - col：表示顶点所在的列号。
  - val：值的数据。
  - right：指向下一个具有相同列号的顶点的指针，用于构建列链表。
  - down：指向该顶点的下一个顶点的指针，用于构建邻接表。
  > 有向图是一种图论中的概念，它由一组顶点和一组有向边组成。每条有向边连接两个顶点，并且有一个明确的方向。与无向图不同，有向图中的边是有方向的，表示从一个顶点到另一个顶点的单向关系。
  > 有向图可以用来表示各种关系和流程，例如电路中的信号传递方向、网站中的链接关系、任务执行的依赖关系等。在有向图中，每个顶点的出度表示从该顶点出发的边的数量，入度表示指向该顶点的边的数量。
  > 有向图的特点是具有方向性和有向边的数量。它可以用来解决一些实际问题，如路径规划、拓扑排序和最短路径等。在计算机科学和网络科学中，有向图是非常重要的数据结构之一。
  > 有向无环图（Directed Acyclic Graph，DAG）是一种特殊的图。
  > AOV-网一定是DAG，即图中不存在环，因为存在环意味着某项活动应以自己为先决条件。
  > AOE-图是DAG。

- **知识点13** 算法必须满足以下准则：输入、输出、有穷性、确定性、可行性

- 输入：一个算法可以有0个或多个输入。数据作为被处理的对象，有时没有输入数据，代表数据已经嵌入到算法中或存储在机器中。
- 输出：每个算法必须要有一个以上的输出数据，它是算法执行后得到的结果。
- 有穷性：一个算法必须可接受的时间内完成，不可以无限的执行下去。
- 确定性：算法中的每一步操作必须有确定的含义，不能有二义性。
- 可行性：算法中的每个操作必须是允许在有限时间内完成的。

- **知识点14** 图的应用算法有克鲁斯卡尔算法、迪杰斯特拉算法、拓扑排序算法

- 克鲁斯卡尔算法：用于解决加权联通图的最小生成树问题。从小到大的顺序选择n-1条边，并保证这n-1条边不构成回路。首先构造一个含有n个顶点但没有边的森林，即每个顶点都是一棵独立的树。然后，按照权的值从小到大的顺序，不断选择边并加入到森林中。在加入每条边时，需要检查是否会形成回路。如果不形成回路，就将该边加入到森林中，否则就舍弃该边。
最终，当森林中所有的树合并成一棵树时，即得到了最小生成树。
- 迪杰斯特拉算法：用于解决从一个顶点到其余各个顶点的单源最短路径问题。它从起始点开始，每次选择当前未访问过的顶点中距离起始点最近的顶点，然后更新该顶点的最短路径。这个过程会不断重复，直到所有的顶点都被访问过，即所有的顶点都被加入到了最短路径中。迪杰斯特拉算法只适用于没有负权重边的图。
- 拓扑排序算法：用于解决一个有向图的顶点排序问题。可以将某一个集合上的偏序关系转化为这个集合的全序关系。
拓扑排序算法首先将图中的所有顶点按照入度为0的顺序加入到一个队列中，然后从队列中取出一个顶点，并将其连接的顶点的入度减1，如果入度为0，则将该顶点加入队列中。重复上述过程，直到队列中没有顶点为止。

- **知识点22** 数据结构是带有结构的数据元素的集合

  相互之间存在一种或多种特定关系的数据元素的集合，精心选择的数据结构可以带来更高的一些运行和存储效率，数据结构一般和高效的检索的算法和索引技术是有关系的。
  数据的逻辑结构和存储结构是两个密切相关的方向，逻辑结构可以对应不存的存储结构。算法设计取决于逻辑结构，而算法的实现取决于存储结构。数据结构的研究内容是构建复杂的软件系统的基础，核心技术是分解与抽象，通过分解可以划分出数据的三个层次，在得到逻辑结构。
  一个具体的问题，到抽象的数据结构过程，通过增加实现细节的考虑，进一步得到存储结构，从而完成计算任务。这是从数据结构到具体实现的过程。

- **知识点24** 存储密度是指数据元素本身所占用的存储量和整个节点结构所占用的存储量之比。

  假设一个单链表的一个结点本身所占的空间为D，指针所占的空间为N，则存储密度为：D/（D+N），一定<1。

- **知识点25** 存储结构由哪两种基本的存储方法实现：

  （1）顺序存储结构：顺序存储结构是借助元素在存储器中的相对位置来表示数据元素之间的逻辑关系，通常借助程序设计语言的数组类型来描述。
  （2）链式存储结构：顺序存储结构要求所有的元素依次存放在一片连续的存储空间中，而链式存储结构，无需占用一整块存储空间。但为了表示结点之间的关系，需要给每个结点附加指针字段，用于存放后继元素的存储地址。所以链式存储结构通常借助于程序设计语言的指针类型来描述。
  （3）散列存储结构：散列存储结构是借助元素的哈希函数值直接定位存储位置的一种存储结构。
  （4）索引存储结构：索引存储结构是建立一个索引表，索引表的每一项包含表中一个记录的地址。

- **知识点26** 数据结构的存储方式有顺序存储、链式存储、散列存储和索引存储
等四种方式。

  （1）顺序存储：通常借助程序设计语言的数组类型来描述。存储密度大、存储空间利用概率高、插入或删除操作效率低。
  （2）链式存储：分为线性和非线性存储，主要体现在顺序映射与分顺序映射两种不同的存储结构。
  顺序存储和链式存储统称为物理结构，在链式存储当中不需要逻辑相邻的结点，之间的关系都是由附加的指针字段表示。
  链式存储结构可以用链接表来表示。
  （3）散列存储：散列存储结构是借助元素的哈希函数值直接定位存储位置的一种存储结构。
  由结点的关键码值决定结点的存储地址，可以用于查找。
  数据存储的方式是一种发展，相比数组，散列的访问方式是高于数组。可以根据数据的部分内容，在散列中找到存储的位置。
  （4）索引存储结构：索引存储结构是建立一个索引表，索引表的每一项包含表中一个记录的地址。定义所有存储的结点在一个区域里。索引存储数是多差平衡数。

- **知识点27** 试述递归程序设计的特点：
（1）具备递归出口。递归出口定义了递归的终止条件，当程序的执行使它得到满足时，递归执行过程便终止。有些问题的递归程序可能存在几个递归出口。
（2）在不满足递归出口的情况下，根据所求解问题的性质，将原问题分解成若干子问题，子问题的求解通过以一定的方式修改参数进行函数自身调用加以实现，然后将子问题的解组合成原问题的解。递归调用时，参数的修改最终必须保证递归出口得以满足。

- **知识点28** 为什么在回溯法、分治法、减治法的算法实现中都要用到栈？
在用回溯法、分治法和减治法解决问题时，算法设计中一般需要用栈结构。
这是因为：回溯法在问题求解过程中，为了实现回溯，需要依次记忆求解的每一步，以便必要时可以按照求解过程的逆次序“回退”。显然要用栈来记忆其求解步骤。用分治法和减治法求解问题时，通常分解后的子问题往往是原问题的较小模式，此时对子问题的解法与原问题的解法是相同的，因此可以用递归的方法来求解，这也就需要用到栈。当然，有时也可以用非递归方法来实现，这种情况下，需要算法设计者建立栈来求解问题。

  - 回溯法：
    （1）回溯法是一种在问题的解空间树中逐层搜索问题解的算法。
    （2）在回溯法中，对于一棵问题解空间树，从根结点出发，深度优先搜索解空间树，直到到达解空间树的叶结点为止。
  - 分治法：
    （1）分治法的基本思想是将一个复杂的问题分成两个或更多的相同或相似的子问题，然后分别求解每个子问题，最后将子问题的解合并得到原问题的解。
    （2）分治法在每一层递归上都有3个步骤：
      （a）分解：将原问题分解为若干个规模较小，相互独立，与原问题形式相同的子问题；
      （b）求解：若子问题规模较小而容易被解决则直接解，否则递归地解各个子问题；
      （c）合并：将各个子问题的解合并为原问题的解。
  - 减治法：
    减治法的基本思想是将一个复杂的问题分解为若干个规模较小，相互独立，与原问题形式相同的子问题，然后分只需要求解其中一个子问题。

- **知识点30** 递归算法比非递归算法花费更多的时间对吗？

- **知识点31** 将两个递增的有序链表合并为一个递增的有序链表。要求结果链表
仍使用原来两个链表的存储空间,不另外占用其它的存储空间。表中不允许有重
复的数据。
  ```c
  void MergeList(LinkList &La,LinkList &Lb,LinkList &Lc)
  { //合并链表 La 和 Lb，合并后的新表使用头指针 Lc 指向
    pa=La->next; pb=Lb->next;
    //pa 和 pb 分别是链表 La 和 Lb 的工作指针,初始化为相应链表的第一个结点
    Lc=pc=La; //用 La 的头结点作为Lc 的头结点
    while(pa && pb)
      {if(pa->data<pb->data){pc->next=pa;pc=pa;pa=pa->next;}
        //取较小者 La 中的元素，将 pa 链接在 pc 的后面，pa 指针后移
      else if(pa->data>pb->data) {pc->next=pb; pc=pb; pb=pb->next;}
        //取较小者 Lb 中的元素，将 pb 链接在 pc 的后面，pb 指针后移
      else //相等时取 La 中的元素，删除 Lb 中的元素
      {pc->next=pa;pc=pa;pa=pa->next;q=pb->next;delete pb ;pb =q;
      }
    }
    pc->next=pa?pa:pb; //插入剩余段
    delete Lb; //释放 Lb 的头结点
  }
  ```
  合并后的新表使用头指针 Lc 指向pa 和 pb 分别是链表 La 和 Lb 的工作指针。初始化为相应链表的第一个结点，从第一个结点开始比较，当两个链表 La 和 Lb 均未到达表尾，则循环执行以下操作：依次摘取 La 和 Lb 当前结点中数据较小者，将摘取来的结点连接到 Lc 表的最后，如果两表中的元素相等，只摘取 La 表中的元素，删除 Lb 表中的元素。确保合并后表中无重复的元素。当一个表到达表层结点，为空时，将空表剩余元素直接在 Lc 表中的最后链接。

- **知识点33** 分治法与减治法的思路有什么相同之处？又有什么不同？
分治法和减治法的共同之处是，它们都是在“分而治之” 思想的指导下发展起来的，基本思路就是把一个规模较大的问题划分为若干个规模较小的子问题，通过对子问题的求解，得到原问题的解。
但分治法和减治法又各自适用于不同的情况，因此它们的求解过程有所不同。用分治法求解的问题，所划分的子问题是互相独立的，且原问题的解需要由各子问题的解合并而成。因此，需要对各子问题分别求解，并合并子问题的解，才能得到原问题的解。可以用减治法求解的问题，虽然也要对原问题进行划分，但因为原问题的解或者只在其中一个子问题中，或者是只与其中的一个子问题的解之间有着某种对应关系，因此只要对相关的一个子问题进行求解，就可以得到原问题的解。当然它也就不存在合并解的过程。可以说，减治法是一种退化了的分治法。

- **知识点34** 设栈中的元素具有什么性质？ 什么情况适合使用栈结构？

  栈是只允许在表的一端进行插入与删除操作的特殊的线性表，表中允许作插入和删除操做的一端称之为栈顶，相应地，另一端称之为栈底。正是因为有这样的限制，使得栈中的数据元素具有“后进先出”的特性。即最先进栈的元素一定是最后出栈的元素。栈的特性决定了栈具有记忆“逆序”的功能，因为栈中元素出栈的顺序就是入栈顺序的逆序。因此，凡是需要记忆“逆序”的情况，在计算机中处理时都可以使用栈结构来实现。比如，要按照与进入时相反的顺序返回，可以把进入步骤顺序入栈，然后按出栈的步骤依次返回即可；又如，拆卸与组装，只要把拆卸步骤顺序入栈，然后按出栈的步骤依次恢复即可。还有象程序的调用与返回，中断的调用与返回等要求按正、逆顺序处理的问题，计算机处理时都是利用栈来解决的。

- **知识点35** 写一个算法，在已知的有序顺序表中插入一个新元素，不改变顺序
表的有序性。
在有序顺序表中插入数据元素需要查找新元素的插入位置并且平移表尾到插入位置的所有元素，查找合适位置和平移数据元素都是重复操作，为了减少循环应该从顺序表中最后的元素开始，依次向前将要插入的元素与表中元素逐个比较同时做需要的向后平移，一旦找到合适的位置则停止重复操作。算法描述如下：

  ```c
  void isertSeqList (SeqList *L, ElemType x ) // 在有序顺序表 L 中插入元素 x，保持表的有序性
  { int j ;
    for (j=L->len；j>0；j-- ) // 从后向前将 x 与表中元素比较并及时移动表中元素
      { if (x>L->list[j-1] ) break ； // 找到位置退出循环
      else L->list[j]= L->list[j-1] ；
      }
    L->list[j]=x ； // 把新元素 x 插入所找到的位置
  }
  ```

- **知识点36** 利用中序线索是否能够实现二叉树的前序遍历？若不能，请说明理
由；
若能，请给出实现的基本思路。
利用中序线索可以实现前序遍历。实现的基本思路是：
从根结点开始前序遍历二叉树。
如果当前遍历的结点有左孩子，则其左孩子就是它的前序后件；
否则，如果当前遍历的结点有右孩子，则其右孩子就是它的前序后件；
不然，一定是叶子结点，此时只要沿中序后件线索查找，直至找到一个有右孩子的结点，该右孩子即为当前所遍历结点的前序后件。一直到查找完全部结点（最后一个结点的前序后件为空），则得到一个前序遍历序列。
